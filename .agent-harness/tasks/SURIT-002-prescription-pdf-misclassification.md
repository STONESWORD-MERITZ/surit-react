# SURIT-002: 처방 PDF 오분류 보정

## 목적
PDF 헤더 OCR 누락 시 처방 PDF가 진료 내역 PDF로 오분류되는 문제를 보정한다.
헤더 신호가 약할 때 페이지 텍스트 신호(처방 관련 키워드/패턴)를 신뢰하여
정확한 PDF 타입 판별이 이뤄지도록 한다.
※ 본 백로그는 PROGRESS.md ③에 남아 있으나 일부 보정이 이미 적용됐을 가능성이 있다.
   1단계로 현재 상태 진단 후, 미흡한 부분만 보강한다.

## 우선순위
P1 (분석 정확도 직결)

## 예상 소요
~30분 (현재 상태가 이미 충분하면 진단 + 회귀 테스트 추가만 ~15분)

## Owner
Cowork (백엔드 파이프라인 코드 수정 작업)

## 범위 (수정 허용 파일)
- `backend/pipeline/pdf_parser.py` (분류 로직)
- `backend/analyzer.py` (필요 시 호출부)
- `backend/tests/test_pdf_parser.py` 또는 신규 테스트 파일

## 범위 외 (수정 금지)
- 프런트 코드 일체
- `backend/filters.py`, `backend/main.py` 등 분류 로직과 무관한 모듈

## 작업 절차
### 1단계: 현재 상태 진단
1. `pdf_parser.py`에서 PDF 타입(진료/처방) 판별 로직 위치 식별
2. 헤더 기반 판별 로직과 페이지 텍스트 fallback 로직 존재 여부 확인
3. 기존 테스트(`test_pdf_parser.py`) 커버리지 확인
4. 진단 결과를 handoff.md Notes에 기록:
   - **이미 충분히 보정됨** → 회귀 테스트만 추가/보강하고 종료
   - **미흡** → 2단계로

### 2단계: 보정 (1단계에서 미흡 판정 시)
1. 헤더 추출 실패 또는 신뢰도 낮을 때, 페이지 본문 텍스트의 처방 키워드/패턴
   (예: "처방전", "조제", "투약일수", "약품명" 등 — 실제 PDF 샘플 기준)으로 fallback
2. 기존 헤더 우선 로직은 유지 (헤더 OCR 성공 시 그 결과 신뢰)
3. 임계값/우선순위는 코드에 주석으로 명시

### 3단계: 회귀 테스트
- 헤더 누락 + 본문에 처방 키워드 → 처방 PDF로 분류되는 케이스
- 헤더 정상 → 헤더 기반 분류 우선 동작 케이스
- 기존 91개 테스트 모두 통과 유지
